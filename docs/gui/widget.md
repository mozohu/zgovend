# GUI 元件類型定義

本文件用於統一 GUI wireframe 中常見元件的命名與用途，適用於 `docs/gui/role-function-pages.mmd` 與 `docs/gui/role-func/*.mmd`。

## 元件類型

### 頂部（Header / Top Bar）
- 定義：頁面最上方固定區塊，承載頁面標題與主要入口。
- 常見內容：返回、標題、搜尋、設定、通知、切換組織。
- 目的：提供頁面定位、主要入口與基礎操作。

### 篩選（Filter）
- 定義：用於快速篩選列表或資料集的條件區。
- 常見內容：日期區間、狀態、類型、區域、距離。
- 目的：縮小資料範圍，提高查詢效率。

### 列表（List）
- 定義：以條列方式呈現多筆資料。
- 常見內容：清單項目、分組、排序。
- 目的：快速瀏覽與定位資料項目。

### 快捷（Shortcut）
- 定義：聚合常用功能的快速入口區塊。
- 常見內容：主要操作、常用功能、捷徑卡片。
- 目的：縮短使用路徑，提升操作效率。

### 卡片（Card）
- 定義：具可視化區塊感的資料呈現單元，通常在列表中或獨立顯示。
- 常見內容：摘要資訊、狀態、關鍵指標、操作入口。
- 目的：在有限空間呈現重點，方便掃描與比較。

### 指標卡（KPI Card）
- 定義：專注於單一指標或數據的卡片。
- 常見內容：數值、單位、趨勢、增減標記。
- 目的：突出關鍵數字，支援決策判斷。

### 指標（Metric）
- 定義：以單行或小區塊方式呈現的關鍵數值或狀態。
- 常見內容：數值、單位、簡短標籤、狀態。
- 目的：在不使用卡片時仍能突出關鍵資訊。

### 圖表（Chart）
- 定義：以圖形視覺化展示趨勢或分布。
- 常見內容：折線、長條、圓餅、堆疊圖。
- 目的：快速理解變化與結構。

### 區塊（Section）
- 定義：頁面中非列表型的資訊區域，用於呈現資訊組合。
- 常見內容：摘要、說明、狀態、操作模組。
- 目的：清楚切分資訊區，提升可讀性。

### 輸入（Input）
- 定義：使用者輸入資訊的欄位或控制元件。
- 常見內容：文字、數字、選單、日期、開關。
- 目的：收集資料或設定操作條件。

### 搜尋（Search）
- 定義：用於關鍵字查詢的輸入區塊。
- 常見內容：搜尋框、搜尋按鈕、清除操作。
- 目的：快速定位目標資料或項目。

### 選項（Option）
- 定義：多個選擇項目的集合，通常為單選或多選。
- 常見內容：下拉選單、核取方塊、單選按鈕。
- 目的：限制輸入範圍並提升一致性。

### 下拉（Dropdown）
- 定義：以展開清單方式提供選項。
- 常見內容：選項清單、已選值、展開箭頭。
- 目的：節省空間並提供明確可選項。

### 開關（Toggle）
- 定義：用於快速切換啟用/停用狀態。
- 常見內容：通知開關、權限開關、功能開關。
- 目的：提供直覺的狀態切換。

### 按鈕（Button）
- 定義：觸發操作或流程的主要入口。
- 常見內容：提交、儲存、匯出、下一步、取消。
- 目的：明確引導使用者動作。

### 操作（Action）
- 定義：針對單一項目或區塊的行為入口集合。
- 常見內容：編輯、刪除、詳情、更多操作。
- 目的：集中行為入口，避免資訊干擾。

### 提示（Hint / Notice）
- 定義：輔助說明或狀態提醒的文字區塊。
- 常見內容：必填提示、風險提示、成功或失敗訊息。
- 目的：降低操作錯誤並提示注意事項。

### 說明（Description）
- 定義：補充背景或操作指引的說明文字區塊。
- 常見內容：步驟說明、注意事項、規則描述。
- 目的：降低理解成本，提升完成率。

### 標題（Title）
- 定義：頁面或區塊的名稱標示。
- 常見內容：頁面標題、區塊標題、子標題。
- 目的：建立清楚的資訊層級與定位。

### 地圖（Map）
- 定義：以地理位置為主的視覺化呈現區域。
- 常見內容：站點標記、範圍、導航路徑。
- 目的：支援位置導向的操作與決策。

### 上傳（Upload）
- 定義：用於附件或圖片的上傳區塊。
- 常見內容：檔案選擇、預覽、刪除。
- 目的：補充證據或輔助資訊。

### 預覽（Preview）
- 定義：在提交前顯示內容或結果的區塊。
- 常見內容：圖片預覽、文件預覽、摘要預覽。
- 目的：降低錯誤提交並確認內容正確。
