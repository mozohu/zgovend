# 給 Max 的外包工作說明書（2026-01-27）

## 角色分工
- Mozo：需求定義、後端實作與 API 提供。
- Max：GUI 設計與 LIFF 前端實作（含與 API 串接）。

## 專案目標
以 LIFF 提供三種角色的行動介面：一般消費者、營運管理員、巡補員，完成客服回報、商品/庫存設定、撿貨與巡補等核心流程。

## 開發範圍（Max）
### 註冊與授權（LIFF）
- 使用者首次加入/開啟時需檢查註冊狀態
- 未註冊者引導 OAuth 註冊（Google / Facebook）
- 完成註冊後才可取得並顯示可用功能權限

### A. 一般消費者（LIFF）
- 客服問題回報（含圖片必填、影片選填）
- 問題單列表/狀態
- 問題單詳情與客服回覆內容查詢（回覆由客服系統提供）

### B. 營運管理員（LIFF）
- 商品主檔管理：列表/新增/編輯/刪除
- 機台管理：列表/詳情/狀態/新增
- 營收與訂單：列表/詳情/報表
- 庫存預約設定（Pre-Replenish）
  - 設定檔清單（可命名、複本、刪除、更名）
  - 內容編輯：貨道商品、滿倉量、庫存量
  - 建立方式：空白建立或從機台現有庫存建立

### C. 巡補員（LIFF）
- 撿貨清單
  - 機台清單多選（作為庫存總量的篩選）
  - 依商品分組彙總：總庫存量/總滿倉量/總缺補量
- 巡補作業
  - 現場掃碼簽到與開始巡補
  - 套用「庫存預約設定清單」
  - 巡補作業表：貨道/商品/滿倉量/存量/缺量
  - 操作：清空/補滿、全補滿/重新整理
  - 結束巡補後才寫入本機庫存

## 介面參考
- 巡補作業畫面需參考 Mozo 提供的現有巡補截圖（外部參考）。

## 交付物
- LIFF 前端程式碼（可建置/部署）
- 對應功能頁面與流程（與規格對齊）
- API 串接（依 mock API 或後端最終 API）
- 基本錯誤/空狀態處理

## 不在範圍內
- 客服系統（電腦網頁管理系統）開發
- 後端 API 開發與資料庫設計
- 機台端系統與硬體整合

## 驗收重點（摘要）
- 功能流程與規格一致（客服回覆查詢、預約設定、撿貨彙總、巡補套用）
- 角色入口與 sitemap 一致
- API 串接可完成主要操作
- 關鍵限制正確：客服回覆為查詢、預約設定不含清空/補滿、結束巡補才寫入
