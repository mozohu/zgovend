# 需求討論紀錄（2026-01-21 18:00）

## 目的與範圍
- 本次討論聚焦於「使用者註冊機制」設計與引導流程。
- 專案目錄目前涵蓋「智購小幫手」之設計與實作，但需要補充說明整體系統層級，以利後續一致性設計。

## 系統層級說明
- 智購小幫手
- 智購販賣機管理大師
- 智購 IOT 管理雲

## 使用者註冊機制需求
- 使用者第一次加入「@智購小幫手」時，後端以 Line UserID 查詢是否已完成註冊。
- 若未完成註冊，系統需引導使用者進行註冊流程。
- 註冊採用 OAuth（Google / Facebook）帳號認證綁定。
- 原因：系統不留存使用者帳號密碼（避免密碼儲存責任與風險）。
- 已完成註冊的 Line 使用者，才可進一步透過「智購販賣機管理大師」進行功能授權。

## 假設與待確認
- OAuth 供應商限定為 Google / Facebook（是否需要可擴充或加入 Apple）。
- Line UserID 與 OAuth 帳號之綁定關係與解除規則。
- 引導註冊的互動方式（例如 LIFF、外部瀏覽器、WebView）。
